# CMPA-101 — 콜키지프리 맵 카드: 카테고리 이미지 채우기 (리서치 + 제안)

> 요청: HTML report 카드의 **빈 이미지 밴드**를 카테고리(예: 족발·곱창)에 맞는
> **상업적 이용 가능한** 이미지/일러스트로 채우기. "가능한지 리서치 후 프로포잘."

## 1. 현황 (왜 비어 있나)
- 카드 상단 사진 밴드(`.photo`, 150px)는 원래 다이닝코드 **대표사진**을 임베드했음.
- **CMPA-87** 에서 보드 지시로 외부 소스 사진·브랜딩을 IP/ToS 위험 때문에 **제거** →
  현재 `find_corkage_free.py:808` 에서 `img = "<div class=thumb></div>"` (빈 회색 밴드).
- 즉 "비어있다"는 건 의도적 제거의 부작용. 다시 **외부 사진을 긁어오는 건 금지**(CMPA-87 회귀).

## 2. 리서치 — 상업적 이용 가능한 이미지 옵션

| 옵션 | 라이선스 | 상업이용 | 귀속(attribution) | 족발·곱창 표현 | 외부요청 | 비고 |
|------|----------|:---:|:---:|:---:|:---:|------|
| **자체 제작 인라인 SVG** | 우리 IP | ✅ | 불필요 | ✅ 직접 그림 | 없음(인라인) | **추천** |
| Google Noto Emoji | OFL1.1 / Apache2.0 | ✅ | 사실상 불필요 | ❌ 범용 🍖만 | self-host | 폴백 후보 |
| Twemoji (Twitter/X) | CC BY 4.0 | ✅ | **필요**(푸터 크레딧) | ❌ 범용만 | self-host | 귀속 부담 |
| OpenMoji | CC BY-SA 4.0 | ✅ | 필요 **+ share-alike(전염)** | ❌ 범용만 | self-host | ⚠️ 상업제품 부적합 |
| 다이닝코드 등 외부 사진 | 타사 IP | ❌ | — | — | 매 요청 | CMPA-87로 금지 |
| Flaticon/Freepik 무료 | 출처별 상이 | △ | 보통 필요·유료 전환 | △ | self-host | 추적 부담·리스크 |
| AI 생성 이미지 | 모호 | △ | — | △ | 생성비용 | 저작권 불명확·spend |

**핵심 발견**
1. 보드가 예로 든 **족발·곱창·막창·장어** 같은 한국 음식은 **이모지/공개 아이콘셋에
   전용 글리프가 없음** → 어떤 이모지 방식을 써도 결국 범용 🍖로 뭉개짐.
   **이걸 식별 가능하게 그리려면 자체 SVG 만이 답.** (보드의 "그려줘" 요청에 정확히 부합)
2. OpenMoji 의 **share-alike(CC BY-SA)** 는 derivative 도 같은 라이선스로 공개 강제 →
   상업 제품에 전염성 위험. 제외 권장.
3. 자체 SVG 는 **인라인**이라 외부 요청 0 → 오프라인 동작·egress 0·프라이버시 우려 0,
   바이트도 작음(아이콘당 평균 ~454B, 16종 검증).

## 3. 추천안 — "자체 제작 SVG 아이콘" (하이브리드 매핑)

- **결정론적 매핑**(classify_major 와 동일 철학, 재크롤 없이 backfill 가능):
  1. 원문 카테고리 **fine 키워드**(족발/곱창/회/초밥/스테이크/장어/훠궈/면/치킨/와인…) — 구체적 우선
  2. **대분류 7버킷** 폴백
  3. 범용 **접시** 아이콘 최종 폴백
- 빈 `.photo` 밴드를 **카테고리 색조 그라데이션 + 중앙 SVG 일러스트**로 교체.
  순위·페어링 배지·미니지도 오버레이는 그대로 유지.
- 전 역(강남·합정·마포·여의도·신논현) backfill 재생성. PDF 경로(md_to_pdf)는 영향 없음(HTML 한정 요청).

  SVG 16종 모두 well-formed XML 검증.

## 4. 가드레일
- 자체 SVG 는 타사 IP 없음 → CMPA-87 회귀 아님, 오히려 공개 가능성 개선.
- 단, 공개/상업 배포 게이트는 **부모 가드레일(CEO 승인)** 영역 — 이번 작업은 내부 R&D 산출물.

## 5. 승인 요청 (택1)
- **A. 자체 제작 SVG (추천)** — 위 POC 방향으로 전 역 적용.
- **B. Noto Emoji self-host** — 범용 음식만, 족발·곱창 식별 불가, 귀속 불필요.
- **C. Twemoji self-host** — 범용 음식만 + 푸터 귀속 크레딧 필요.

> 승인 시: 매핑 사전 보강(미매칭 카테고리 점검) → `find_corkage_free.py` 통합
> → 전 역 backfill 재생성 → diff/스모크 검증.
