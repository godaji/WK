# whisky-synonyms — 위스키 동의어/표기변형 정의 (CMPA-22)

크롤링·OCR·ASR로 들어온 **"같은 술의 다른 표기"** 를 `whisky-list.csv` 의 정본 id 로
모으기(dedup) 위한 자산. CEO 지적사항 — `"러셀 리저브 싱글 베럴"` 과 `"러셀 리저브 싱글 배럴"`
이 서로 다른 상품으로 잡히던 문제 — 를 해결한다.

## 구성 파일

| 파일 | 역할 |
|---|---|
| `assets/whisky-synonyms.yaml` | **동의어 정의의 정본(source of truth)**. 토큰 동의어 + 잡음 패턴 + 상품 매칭규칙 + 비위스키 제외목록. 사람이 직접 편집한다. |
| `scripts/normalize_whisky_name.py` | 위 yaml을 읽어 `canonicalize(raw)→whisky-list id` 를 수행하는 실행기. 단건/전수(audit) 모드. |
| `assets/whisky-aliases.csv` | **전수 조사 결과**(생성물). 원시표기 → 정본 id 전수 매핑. 크롤 단계에서 dedup 룩업 테이블로 바로 사용 가능. |

## 사용법

```bash
# 단건 정규화
python3 scripts/normalize_whisky_name.py "러셀 리저브 싱글 베럴 750ml"
#  -> [matched] w077 러셀스 리저브 싱글배럴

# 전수 조사: 모든 한글 표기를 정규화 → whisky-aliases.csv 재생성 + 요약 리포트
python3 scripts/normalize_whisky_name.py --audit
```

크롤 파이프라인 연동 예:
```python
from scripts.normalize_whisky_name import Normalizer, load_rules
norm = Normalizer(load_rules())
r = norm.canonicalize(scraped_name)   # {'status','id','name_ko',...}
if r["status"] == "matched":
    price_row["whisky_id"] = r["id"]          # 정본 id 로 dedup/집계
elif r["status"] == "excluded":
    drop(scraped_name)                        # 비위스키(보드카/진/꼬냑 등)
else:
    queue_for_review(scraped_name)            # 정본 미등록 후보
```

## 정규화 파이프라인 (4단계)

1. **token_synonyms** — 상품 불문 공통 표기/철자/음차 통일 (긴 변형 우선 치환)
2. **noise_patterns** — 용량/포장/가격/대괄호 등 식별 무관 잡음 제거
3. 공백 정리
4. **products** 의 `match`(모두 포함) / `not`(하나라도 있으면 제외) 토큰 부분일치 — 위→아래, 먼저 맞는 정본 채택

## 동의어로 흡수한 대표 변형 (전수 조사에서 실제 관측)

- **배럴 ↔ 베럴**, **캐스크 ↔ 케스크**, **셰리 ↔ 쉐리**, **몰트 ↔ 모트**, **싱글몰트 ↔ 싱글트/싱글 모트**
- **버번 음차오류**: `법원`→버번, `버유스키/법원위스키/버뉴스키`→버번위스키 (ASR/OCR)
- **가쿠빈 ↔ 각쿠빈**, **아부나흐 ↔ 아브나흐/아브나오/아브나르/아브나우/아브나후** (Aberlour A'bunadh, 음차 6종)
- 브랜드 음차/띄어쓰기: `글랜그란트/더 글렌 그란트`→글렌그란트, `글랜드로나/글랜드 로낙`→글렌드로낙,
  `골든 블루`→골든블루, `글램 모렌지`→글렌모렌지, `조니 워커/조니워커블`→조니워커, `뱅콜`→뱅크홀,
  `포로지 지스/호로지스`→포로지스, `잭다니엘스/잭다니의/잭다니에스`→잭다니엘
- 조니워커 라벨: `더블랙`→더블블랙, `블랙라베`→블랙라벨, `블루라/블로라`→블루라벨, `누루비/누비`→루비, `그인라`→그린라벨
- **짐빔 ↔ 진빔/진빈** (단, `진`=gin·`바톤진`은 비위스키로 제외)

## 전수 조사 결과 (`--audit`, 2026-05 기준)

- 원시 표기 **distinct 640종** → **matched 500** (= 정본 88 상품), **excluded 14**(비위스키), **unmatched 126**

## 유지보수

- 새 표기/오타가 들어오면: 공통 철자변형이면 `token_synonyms`, 특정 상품 한정이면 해당 product의 `aliases_exact` 에 추가 후 `--audit` 재실행.
- 새 정본 상품을 `whisky-list.csv` 에 추가하면, 같은 id의 매칭규칙을 `products` 에 추가한다(구체적 규칙일수록 위에).
