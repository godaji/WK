# 데일리샷 최저가 수집기 (CMPA-19)

데일리샷(dailyshot.co)에서 **이마트·트레이더스·코스트코** 추적 위스키의 `(수집일, 위스키명, 가격)` 최저가를 수집한다.

## 실행

```bash
python3 pipelines/dailyshot/crawl_dailyshot.py [수집일=오늘] [출력CSV]
```

- 산출: `data/whisky-prices/2026-05_dailyshot.csv` (UTF-8 BOM)
- 메트릭: `data/whisky-prices/_dailyshot_metrics.json`

## 데이터 소스

런타임 디스커버리로 확정한 데일리샷 공개 검색 API:

```
GET https://api.dailyshot.co/items/search/?q=<위스키명>&page=<n>
-> { count, next, results:[ { name, top_product_id, price, discount_percent, category, ... } ] }
```

데일리샷은 마켓플레이스라 동일 상품(`top_product_id`)이 셀러별로 여러 가격으로 노출된다.
→ 매칭 상품군의 **`price` 최저값 = 그 위스키의 데일리샷 최저가**. `price`는 이미 할인 반영된 노출가.

## 대상 리스트

`data/whisky-prices/{2026-03,04,05}.csv` 의 `술이름` 중 `위치`가 **이마트/트레이더스/코스트코** 인 항목만
자동 로드(하드코딩 없음) → 용량/가격잡음/패키지어 정제 + 중복제거. 롯데마트 등은 과업 범위 밖이라 제외.

## 매칭 정책 (정밀도)

- 검색결과 중 `category == '위스키'` 만 채택(동명 코냑/진 배제).
- 500ml 이하 소용량·미니어처·전용잔 패키지는 병당가 왜곡 → 최저가 후보에서 제외(표준 700/750ml·1L만).
- 대상명의 핵심 토큰이 상품명(정규화)에 **모두 부분문자열로** 존재해야 매칭
  → 띄어쓰기 차이(`더블캐스크` ↔ `더블 캐스크`)에 견고.
- 상품명 추가글자 최소(=plain 우선) 군에서 최저가. 추가글자 0 → `정확`, 그 외 → `근접`(다른 버전 가능, 스팟체크 권장).

## 한계 / 정직성 노트

- **MISS 대다수는 상류 OCR 소스명 깨짐**(예: `글랜드로`, `잭다니의`, `조니워커 그인라`)이 원인 — 수집기 결함이 아니라 타깃 리스트 품질 문제.
- 단일 브랜드 조각 질의(`제임슨` 등)는 모호 → `근접`으로 표기. `정확` 행은 신뢰, `근접` 행은 검토 후 사용.
- 저빈도(요청 간 0.7s 슬립), 인증우회 없음, 키워드 질의만(카탈로그 통복제 X).
