# ocr_poc — @whiskeykey 화면 오버레이 OCR 타당성 스파이크 (CMPA-172)

> **결론: GO (타당함).** @whiskeykey 영상은 **합성 그래픽 오버레이**(제품명·가격을
> 또렷한 흰 글씨로 화면에 박아 줌)를 쓰기 때문에, **스샷→OCR 로 이름·가격 추출이 잘 된다.**
> 이는 CMPA-5 의 OCR NO-GO(싼디의 *촬영된 매대 종이태그*, 이름 garbled·full-row 55%)와
> **완전히 다른 케이스**다. 합성 오버레이는 고대비·고정위치·일관 폰트라 OCR 친화적.

내부 R&D 타당성 측정용 스파이크. **운영 적재/공개 배포 아님** (CMPA-7 게이트 유지).

## 무엇을 측정했나
대상: `@whiskeykey` 2026-06-01 영상 `dMF5i15ucJQ`("판매중인 모든 트레이더스 위스키 가격 정보"),
8분 02초, 720p. 같은 영상의 **ASR 적재본(2026-06.csv, 61행)** 을 비교 기준으로 삼음.

오버레이 레이아웃(샘플: `sample_frame_clean.jpg`):
- **상단 우측**: 제품명 KR + EN + 용량 (예: `1792 스몰배치 750ml` / `1792 SMALL BATCH`)
- **우측 중단**: 정상가(취소선) → 적립할인액 → **최종가(가장 큰 폰트)** (예: 44,980 / -5,000 / **39,980**)
- **하단 좌측**: `촬영 일자: 2026.6.1` / `촬영 장소: 트레이더스 동탄점`

## 측정 결과 (하드 넘버)
| 지표 | 결과 |
|---|---|
| OCR 엔진 | PaddleOCR 3.6.0 `lang=korean` (`enable_mkldnn=False` 필수, CMPA-6 교훈) |
| 프레임 추출 | 3초당 1프레임 → 161프레임 (yt-dlp 720p 51MB + ffmpeg) |
| OCR 속도 | name+price 크롭 2장/프레임, **~6초/프레임** (161프레임 ~16분, CPU) |
| **제품명 정확도** | **~26–27 / 30 세그먼트 깨끗·정확** (잭→책 등 미세 자모오인만). ASR 대비 **대폭 개선** |
| **가격 숫자 OCR** | **사실상 완벽** — 읽힌 숫자열은 ASR 와 정확히 일치(28,980·27,980·54,800·25,980…) |
| 최종가 *선택* 휴리스틱 | 매칭 13쌍 중 6 일치(스파이크용 거친 휴리스틱). **불일치 7건은 전부 '최종가 선택' 실패** (1L·라인분할로 `100ml당 단가`가 새는 케이스). **OCR 오독 아님** |

### 핵심 발견
1. **이름**: ASR 의 음차 garbled(`글램피닉 15년`, `조니어 블랙 루비`, `글랜드러 오드더 엠버스`)가
   OCR 에선 `조니워커 블랙 루비 700ml`처럼 **깨끗**하게 나온다 → **OCR 가 이름 품질을 크게 올림.**
2. **가격**: 오버레이 숫자는 1.00 신뢰도로 또렷이 읽힌다. 남은 과제는
   "정상가/할인액/최종가/100ml단가 중 무엇이 최종가인가"라는 **필드 선택 로직**뿐 —
   OCR 능력 한계가 아니라 풀 수 있는 엔지니어링.
3. **보너스**: OCR 는 ASR 가 못 주는 **구조화 필드**(정상가·할인액·최종가·용량·촬영일·촬영장소)를
   한 화면에서 동시에 준다.
4. **전환 프레임 주의**: 위스키가 바뀌는 애니메이션 프레임은 180° 회전 텍스트라 OCR 가 깨진다
   (`sample_frame_transition.jpg`). → 세그먼트 중앙의 안정 프레임만 채택하면 됨.

## 재현 방법
```bash
ID=dMF5i15ucJQ
yt-dlp -f 136 -o vid.mp4 "https://www.youtube.com/watch?v=$ID"   # 720p video-only(51MB, merge 불필요)
ffmpeg -i vid.mp4 -vf fps=1/3 frames/f%03d.png                   # ffmpeg 없으면 imageio_ffmpeg 번들 사용
python3 ocr_dump_frames.py frames raw.json                       # 1단계: 프레임→OCR 박스 JSON (~16분)
python3 extract_overlay_fields.py raw.json                       # 2단계: 이름/최종가 추출+dedup (즉시)
```
동봉: `raw_dMF5i15ucJQ.json`(원본 OCR 박스), `poc_result_dMF5i15ucJQ.csv`(30세그먼트 산출),
`sample_frame_clean.jpg` / `sample_frame_transition.jpg`(증거 프레임).

## 후속 구현 시 과제 (운영화 go 결정 시)
- **최종가 선택**: 최대폰트 + 최종가 전용 y-밴드 절대좌표 타게팅, `100ml당` 단가·취소선 정상가 배제
  (OCR 라인분할 대비 박스 인접도 기준 병합).
- **세그먼트 분할**: scene-change 로 위스키당 1 안정 프레임만 샘플 → 프레임 수·OCR 시간 1/3 절감.
- **이름 정규화**: 기존 `scripts/normalize_whisky_name.py` 로 정본 id 매칭 (ASR 파이프라인과 동일).
- **위치 표기**: 오버레이엔 `트레이더스 동탄점`이라 찍히나, 보드 규약상 `트레이더스` 로만 적재(지점명 제거).
- **활용안**: OCR 를 @whiskeykey **1차 소스 또는 ASR 교차검증**으로 — 이름은 OCR, 가격은 OCR(정확)·
  ASR(백업). ASR 의 `낮음`(자릿수 누락) 플래그를 OCR 가 교정 가능.
- **거버넌스**: 영상 다운로드는 내부 R&D 만(다운로드→측정, 미공개). 공개 surface 적재는 CMPA-7 게이트.
