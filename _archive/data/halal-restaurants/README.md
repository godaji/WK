# `data/halal-restaurants/` — 무슬림(할랄) 식당 데이터셋 (CMPA-81 / CMPA-86)

> 무슬림용 식당 맵 **데이터셋(csv)** 의 정본. 엔진: [`pipelines/halal_restaurants/`](../../pipelines/halal_restaurants/)
> 콜키지프리맵과 **분리된 독립 제품 라인**(CEO 지시). 폴더구조는 `corkage-free`·`whisky-prices` 규약과 동형.

## 폴더 분리 (보드 지시 — corkage-free 와 동일)
- **데이터셋(csv)** = 파이프라인이 소비 → 이 디렉터리 `data/halal-restaurants/`.
- **사람이 보는 리포트(md/html)** = [`reports/halal-restaurants/`](../../reports/halal-restaurants/).
- 각 디렉터리에 `_runs/` 날짜 스냅샷(`{stem}__run<YYYY-MM-DD>.csv`, 주간 재실행 감사추적, CMPA-38/45).

## 1. 산출물

| 파일 | 위치 | 내용 |
|------|------|------|
| `{역명}_무슬림식당.csv` | `data/halal-restaurants/` | **정본 데이터.** 역 도보권 무슬림 식당 (A·명시 + B·추정), `utf-8-sig`(BOM) |
| `{역명}_무슬림식당.md` | `reports/halal-restaurants/` | 사람이 읽는 표 + 대분류 분포 + 고지문 |
| `{역명}_무슬림식당.html` | `reports/halal-restaurants/` | **카드뷰**: 등급 배지·근거·주의·면책 배너 + 등급×음식 필터 |
| `{name}_무슬림식당_공공.csv` | `data/halal-restaurants/` | **공공 오픈데이터 ingest 정본**(Tier A 권위, `출처`=발급기관). `ingest_opendata.py` 산출 |

현재(2026-05-31): `이태원역_무슬림식당` (A 25 / B 25, 정제 후) · `안산역_무슬림식당` (A 2 / B 13).

## 2. CSV 컬럼 사전 (맵 마커 조인키 ★)

```
순위, 할랄등급★, 식당명, 카테고리, 대분류★, 근거, 주의, 출처★,
도로명주소, 전화, 도보거리_m, 도보_분, 평판점수, user_score, 리뷰수,
대표사진, 네이버지도, 식당ID, lat★, lng★
```
> CMPA-102: 외부 소스 브랜드 흔적 제거 — `다코점수`→`평판점수`(인기·평판 종합점수, 정렬키),
> `다이닝코드링크`(프로필 URL)→`식당ID`(rid 단독, 내부 조인키), `대표사진`은 비움(HTML 카드는
> 사진 대신 **역↔식당 위치 지도**를 표시), `출처`의 `DiningCode`→`공개검색 추정`.

| 컬럼 | 맵 활용 |
|------|---------|
| `할랄등급` ★ | `A·명시할랄`/`B·추정(확인필요)`(공개검색 추정) · `A·공인할랄`/`A·자가인증`/`B·무슬림프렌들리`/`B·포크프리`(공공데이터) — **마커 색/필터 차원** |
| `근거` | 왜 이 등급인지(명시신호 / 할랄친화 cuisine / 해산물·채식 / KTO 분류) — 팝업 투명성 |
| `주의` | "매장 확인 필수" 고지문 — **전 행 필수 노출** |
| `출처` ★ | `공개검색 추정`(내부R&D) vs 발급기관명(예 `한국관광공사`=Tier A 권위·재게시 가능) — **권위 구분** |
| `lat`/`lng` ★ | 마커 좌표(역↔식당 위치 지도·마커) |
| `대분류` ★ | 중동·인도(할랄친화)/물고기·해산물/채식/… — 필터 |
| `식당ID`/`네이버지도` | 내부 조인키(rid) · 네이버 지도 딥링크 (`대표사진`은 미사용) |

## 3. 등급 의미 (요약)
- **A·명시할랄** = 이름/카테고리에 할랄·무슬림·이슬람 명시. (그래도 인증범위·갱신·도축은 확인 권장)
- **B·추정(확인필요)** = 할랄친화 cuisine 또는 해산물·채식 업종 추론. **할랄 보증 아님.**
  돼지파생물·조리용 알코올·자바이하 도축은 매장 확인 필수.

## 4. 재생성
```bash
python3 pipelines/halal_restaurants/find_halal.py --station 이태원역 --radius 800
```
→ csv 는 `data/halal-restaurants/`, md/html 은 `reports/halal-restaurants/` 에 생성.

## 5. ⚠️ 법무/소싱 가드레일
공개 검색 신호 기반 추정 = **내부 R&D** 범위. 공개/상업 surface 재게시는 콜키지맵과 동일하게
**CEO 법무·소싱 승인 필요**. 본 설계의 다음 단계 = 정부 공공 오픈데이터(data.go.kr 무슬림친화
음식점)를 **Tier A 권위 정본**으로 ingest(재게시 가능 라이선스)하여 공개 검색 추정 의존을 낮추는 것.
종교 식이 정보 특성상 공개 시 **면책 고지**(추정·할랄 미보증·인증/매장확인 권장) 필수.
