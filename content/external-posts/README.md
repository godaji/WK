# 외부 작성 글 받는 곳 (external-posts) — CMPA-353

여기는 **CaskCode 외부에서 작성한 블로그 글(.md)을 떨궈 두는 폴더**입니다.
보드가 이 폴더에 마크다운 파일을 넣어 두면, 에이전트가 **검증 → 블로그에 배포**합니다.
(글 작성은 외부에서, **우리는 배포만** 합니다.)

## 쓰는 법 (보드)
1. 아래 [프런트매터 계약](#프런트매터-계약)대로 `.md` 파일 1개 = 글 1개로 만든다.
2. 파일 이름을 **`YYYY-MM-DD-영문슬러그.md`** 로 짓는다. (예: `2026-06-20-nvidia-thesis.md`)
   - 날짜는 발행일, 슬러그는 URL에 쓰일 영문(소문자·하이픈).
3. 이 폴더(`content/external-posts/`)에 넣고, 이슈에 "배포해줘" 한마디 남긴다.
   - (후속) 폴더 감시 routine 을 붙이면 자동 배포로 전환 가능.

## 에이전트가 하는 일 (배포 파이프라인)
1. 프런트매터 검증 — 필수 키·`categories` 가 **렌더 버킷**인지 확인
   (잘못된 카테고리는 글이 어디에도 안 보이는 고아글이 됨 — CMPA-326 규칙).
2. `blog-md/_posts/<같은이름>.md` 로 이관.
3. `caskcode-publish` surgical push 로 라이브 반영 → `curl | grep` 으로 라이브 확인.
4. 이슈에 라이브 URL 보고. 처리한 원본은 이 폴더에서 `published/` 로 이동(이력 보존).

## 프런트매터 계약
파일 맨 위에 아래 블록을 그대로 채운다. (`---` 두 줄 사이)

```yaml
---
layout: post
title: "[데이터] 글 제목"
date: 2026-06-20 21:00:00 +0900
categories: [data]
tags: [데이터분석]
robots: index,follow
---
```

| 키 | 필수 | 설명 |
|---|---|---|
| `layout` | ✅ | 항상 `post` |
| `title` | ✅ | 독자에게 보이는 제목 |
| `date` | ✅ | `YYYY-MM-DD HH:MM:SS +0900` (KST). 파일명 날짜와 맞출 것 |
| `categories` | ✅ | 렌더 버킷 택1 — 개발글 `[dev]`, 데이터분석 `[data]`, 시음 `[tasting]`, 위스키 시세 `[wprice]` |
| `tags` | ⬜ | 자유 태그(분류엔 영향 없음) |
| `robots` | ⬜ | 색인 정책. 미기재 시 사이트 기본값 |

> ⚠️ `categories` 가 `dev`/`data`/`tasting`/`wprice` 등 **템플릿이 렌더하는 버킷**이 아니면
> 글 URL은 200이어도 어느 목록에도 안 걸리는 **고아글**이 됩니다.

## 본문 작성 팁
- 본문은 일반 마크다운(GFM). 표/이미지/링크 OK.
- **이미지**: `![설명](URL)` — 데스크톱에서 480px 로 자동 캡(모바일은 100%), CSS 전역 적용이라 따로 안 해도 됨.
- **표**: 모바일(≈360px) 우선 — 컬럼 많으면 깨짐. 가능하면 2컬럼, 넓으면 가로 스크롤됨.
