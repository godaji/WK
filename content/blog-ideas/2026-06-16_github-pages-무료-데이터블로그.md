# 서버 비용 0원으로 데이터 블로그 굴리기 — GitHub Pages + 정적 사이트 스택

> 오늘의 블로그 아이디어 · 2026-06-16 · 기둥: 💻 **Code** · 작성: CaskCode(CMO)

## 한 줄 요약
매일 가격 데이터를 긁어 글을 올리는 블로그를, **서버 한 대 없이** GitHub Pages
정적 사이트로 운영하는 법. 우리(CaskCode)가 실제로 쓰는 스택과 비용 구조를 그대로
공개한다 — 마크다운 → 정적 HTML → 무료 호스팅, 데이터 갱신은 푸시 한 번.

## 왜 이 글인가 (가치-대비-가격 렌즈)
- 06-14 'Paperclip 에이전트 분업' 글이 **누가(에이전트) 굴리나**였다면, 이 글은
  **무엇 위에서(인프라) 굴리나** — 같은 시리즈의 다른 층이라 중복이 아니라 보완.
- 우리가 자랑할 자산이 바로 **이 블로그 자체**다. 라이브로 돌아가는 정적 스택을
  그대로 해부하면 한계비용 0 — 새로 만들 게 없고 신뢰도 높다("실제로 이렇게 굴린다").
- "데이터 블로그 = 비싼 서버·DB"라는 오해를 깬다. 우리 운영비는 **사실상 0원**
  (호스팅 무료, 빌드 로컬/CI). 이 '가성비' 프레임이 우리 브랜드 톤과 정확히 맞는다.
- 따라 하기 쉬운 실전 레시피라 **유입·체류·북마크**에 강하다(개발 검색 트래픽).

## 독자
**개인 블로그·사이드프로젝트를 무료로 띄우고 싶은 개발자/데이터 취미러.** 정적
사이트 제너레이터를 들어는 봤지만 "데이터가 매일 바뀌는 블로그도 정적으로 되나?"가
궁금한 사람. 서버·DB 비용이 부담돼 시작을 미룬 사람.

## 구성 (초안 목차)
1. **왜 정적인가**: 매일 갱신되는 가격 데이터 블로그인데도 DB·백엔드가 없는 이유
   (읽기 전용·캐시 친화·공격면 최소·비용 0).
2. **스택 한 장 요약**: 마크다운 글 + 데이터 CSV/JSON → 빌드 스크립트로 정적 HTML →
   GitHub Pages 호스팅. 각 조각이 무료인 지점 표기.
3. **데이터가 바뀌면?**: 수집 파이프라인이 데이터 갱신 → 페이지 재생성 → 푸시 한 번
   = 배포. "글은 사람이, 데이터는 자동" 분리.
4. **모바일 우선·표 깨짐 같은 실전 함정**: 좁은 화면(360px) 검증, 표 2컬럼화 등
   우리가 실제로 부딪힌 것 몇 가지(담백하게).
5. **비용 정산**: 월 운영비를 항목별로 — 호스팅/도메인/빌드. "여기까지가 0원, 여기부터 돈."
6. **시작 체크리스트**: 빈 레포 → 첫 글 → 무료 배포까지 최소 단계.

## 우리가 가진 근거 / 소스
- 라이브 블로그(CaskCode) 자체가 GitHub Pages 정적 사이트로 운영 중 —
  배포 경로/빌드 흐름은 [[caskcode-publish-deploy-path]]에 정리돼 있음(소스 기반, 추정 아님).
- 빌드 제너레이터(`pipelines/shilla_dutyfree/build_blog_md.py` 등)와 CSS가
  실제 산출물을 만든다 — 글에 인용할 구조가 코드에 그대로 있다.
- 모바일 우선·표 2컬럼·이미지 max-width 등 실전 함정은 CLAUDE.md 운영 원칙(CMPA-255/292)에
  사실로 기록됨.
- **주의**: 무료 한도·가격은 플랫폼 정책이라 **글에 "○○년 ○월 기준"** 단서 필수
  (호스팅 무료 한도는 바뀔 수 있음 — 발행 시점 공식 문서 1차 확인 후 게재).
