# 블로그 아이디어 (Blog Ideas) — CMPA-252

매일 아침 CMO(CaskCode)가 **블로그 아이디어 1개**(약 1페이지)를 제안하고
`shhong@dudaji.com` 으로 이메일을 보낸다. 제안은 이 폴더에 날짜별 md로 누적한다.

- **파일명**: `YYYY-MM-DD_<짧은-슬러그>.md` (날짜 prefix로 정렬 → 최신이 위/아래로 모임)
- **기둥(pillar)**: 🥃 Cask(위스키·가성비·면세·시음) 또는 💻 Code(개발·데이터·Paperclip)
- **원칙**: 담백한 카피([[블로그 카피 톤 원칙]]), 저자 호칭은 `CaskCode`,
  사실·데이터 중심, 가격·가용성 주장엔 출처·수집일.
- **루틴**: 매일 아침 07:00 KST 자동 실행 → 새 제안 생성 → **Todo 아이디어 이슈 생성**
  (미배정 — 보드가 검토·선별) → 이메일. 헬퍼: `scripts/create_blog_idea_issue.py`(이슈),
  `scripts/send_blog_idea.py`(메일). 모든 이슈는 CMPA-252(허브) 자식으로 쌓인다.
- **글은 쓰지 않는다**(보드 지시 2026-06-09): 아이디어 '제안'까지만. 실제 블로그 글/초안은
  보드가 골라 진행을 지시할 때만. 이슈는 Todo·미배정으로 둬 자동 작업되지 않게 한다.
- **중복 방지**: 새 아이디어를 내기 전 아래 '발행된 제안 목록'을 확인해
  이미 제안한 주제와 겹치지 않게 한다. 새 제안을 추가하면 목록에 한 줄 기록한다.
